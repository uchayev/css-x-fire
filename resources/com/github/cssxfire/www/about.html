CSS-X-Fire (1.38)

Requirements

This plugin requires Firefox with Firebug.

Users of Google Chrome or Firefox with Firebug Lite, please look here.

Get connected

Download and install the corresponding browser plugin - cssxfire.xpi (1.37) - to connect Firebug CSS editor with the IDE plugin.

Using

Use Firebug CSS editor to edit and add style properties.

The IDE toolwindow shows the changes made in Firebug in a tree view, categorized by file and CSS selector. From here you can apply or discard changes on an entire file, selector or on a specific property.

The property nodes - the tree leafs - will either be blue (modified), green (new) or gray (invalid). They are also navigatable, e.g. middle-click, double-click or press enter key on a valid property to jump to the corresponding highlighted source code.

Settings

The settings are stored on a project level. Here is a brief explanation of the parameters and what they do:

Clear pending changes when leaving or reloading page: When checked, all pending changes will be cleared whenever the web page is refreshed or left, or when Firebug is (re)opened.

Match CSS3 media queries: When checked, all incoming changes will be filtered down to the selectors matching the media query.

Currently opened files: When checked, all incoming changes will be filtered down to the files that are currently opened in the editor.

Match filename: When checked, all incoming changes will be filtered down to files with a the given filename provided by Firebug, regardless of path. Note that this setting will have no effect when using route mappings (see below).

Use routes: When checked, each incoming change will be routed to one single project file, based on the mappings table. Note: These routes are only used by CSS-X-Fire. A route that ends with whitespace or does not start with a forward slash will not be able to be added.
